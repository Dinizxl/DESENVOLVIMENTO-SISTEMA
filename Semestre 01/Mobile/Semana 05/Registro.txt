1. Lógica do Catálogo de Produtos
Para passar o produto da tela principal para a de detalhes, o melhor é enviar apenas o ID do produto.
 
Como funciona: Quando o usuário clica no produto, o app manda o código (ID) dele para a próxima tela.

Na tela de detalhes: O app pega esse ID e faz uma busca no servidor para mostrar as informações mais novas (preço, fotos e descrição). Isso evita erros e deixa o app mais rápido.

3. Requisições HTTP e Dados (Tarefa: "Comprar leite")
 
Para buscar tarefas: Uso o método GET na URL do servidor.

Para enviar uma tarefa: Uso o método POST e mando o texto da tarefa no "corpo" da mensagem.

Exemplos de como o dado é escrito:
 
Em JSON (mais comum):

{ "tarefa": "Comprar leite" }
 
Em XML (mais antigo):

<item>
  <tarefa>Comprar leite</tarefa>
</item>
